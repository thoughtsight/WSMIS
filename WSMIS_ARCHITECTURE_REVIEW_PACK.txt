2.4 PAGE ROUTING LOGIC
------------------------

    1. Get current page from session state
    2. Validate route exists in RouteRegistry
    3. Check admin route protection (?admin=true)
    4. Lazy-import page render function
    5. Call safe_render() wrapping the render function
    6. Render UniversalFooter

  Page Capabilities (app.py lines 50-69):
    - default_period: "1M" or "3M"
    - comparison_mode: True/False
    - show_period_filter: True/False
    - show_comparison_filter: True/False
    - show_service_type_filter: True/False
    - additional_module_filters: ["Advisor"], etc.

3.2 ROUTING — Route Registry & Validation
-------------------------------------------

  services/route_service.py:

  class RouteType(Enum): PUBLIC, PROTECTED, ADMIN

  @dataclass Route:
    path, type, title, handler, allowed_roles, allowed_dashboards, query_params

  class RouteRegistry:
    _routes: Dict[str, Route]
    _initialize_default_routes()  # Registers all 20+ routes
    register_route(route)
    is_valid_route(path) -> bool
    get_route_type(path) -> RouteType

  class RouteValidator:
    validate_route(path) -> (bool, error)
    validate_query_params(path, params) -> (bool, error)
    check_route_access(path, role) -> (bool, error)

  class RouteService:
    registry: RouteRegistry
    validator: RouteValidator
    navigate_to(path, params)
    sync_url_to_session()

  Performance:
    - Aggregation cache is in-memory (per-session, not shared)
    - No persistent caching across sessions
    - Streamlit rerun triggers full data reload on filter changes
